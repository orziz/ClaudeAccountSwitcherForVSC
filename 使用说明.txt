Claude 多账号 · 环境启动台 —— 使用说明
==========================================

它解决什么
----------
让你同时开多个 VSCode 窗口，每个窗口登不同的 Claude 账号，并行跑、互不影响。
适合：不同项目用不同账号（工作 / 外包 / 开源 / 个人……）。

核心概念
--------
“环境” = 一个独立账号的一套独立配置（CLAUDE_CONFIG_DIR，自带各自的登录态）
        + 一套独立的 VSCode 数据（--user-data-dir，逼出独立进程，环境变量才隔离得开）。
选中环境点“打开窗口”→ 启动一个绑定该账号的独立 VSCode 实例。
多个环境可同时开着，各用各的号、各管各的额度，谁也不覆盖谁。

============================ Windows ============================
打开：双击桌面“Claude 多账号启动台”，或双击 launch.vbs。
绿色可携带：整个文件夹拷到哪都能跑（路径自动探测）。

新建环境：点“新建环境...”→ 起名 → 选默认项目目录(可跳过) → 选绑定方式：
  - 现场登录        ：开窗后 /login 一次，永久记住
  - 用当前登录的号  ：把当前默认登录的号直接灌进去，免再登
  - 从已存账号档灌入：复用旧切号工具存过的快照
日常：选中环境 →“打开窗口”。● = 已登录，○ = 未登录。
“单窗口切号(旧)”按钮 = 打开旧的文件交换式切号工具（二合一入口）。

============================ macOS =============================
首次：终端里 cd 到本文件夹，执行  chmod +x claude-env-launcher.command
打开：双击 claude-env-launcher.command（会弹原生对话框，零安装）。
读 JSON 用系统自带 plutil，弹窗用 osascript，不装任何东西。
若提示找不到 code：在 VSCode 里运行
  “Shell Command: Install 'code' command in PATH” 一次即可。

跨机器携带建议
--------------
拷“脚本”过去即可；环境(envs)建议在每台机器各自新建。
凭据 JSON 跨系统通用，但 vscode-userdata 是各系统自己的，别跨系统混用。

文件清单
--------
ClaudeEnvLauncher.ps1        Windows 启动台（桌面图标指向它）
claude-env-launcher.command  macOS 启动台
claude-switch.ps1            旧“单窗口切号”工具（保留备用）
launch.vbs                   Windows 静默启动器
envs\<环境名>\               每个环境一套（含登录令牌，敏感！勿外传、勿提交 git）

注意
----
- 每个环境是独立 VSCode 实例，首次打开是“干净”界面（默认设置、默认主题）。
- envs 里存的是各账号的登录令牌，请按敏感数据对待。
